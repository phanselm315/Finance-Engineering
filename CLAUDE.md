# Repo instructions — Claude Architect Journey

This is Peter's public AI-build portfolio. Top-level `README.md` is the pitch; each project
lives at `projects/NN-slug/README.md`; `LOG.md` is the chronological record;
`TOOLSTACK.md` is the living inventory of what I build with.

## When running a status sweep (architect-journey-update)

In addition to surveying and updating the project READMEs, **review and refresh
[`TOOLSTACK.md`](./TOOLSTACK.md)**: add any new connector, skill, model, or framework that
entered the rotation since the last sweep, and bump its *Last updated* date. Treat it as a
first-class part of the sweep, not an afterthought.

## Voice

First person, direct, concrete. Honest about negative results and pauses. No client, parish,
or employer-traceable names beyond what existing READMEs already carry. No corporate filler.
